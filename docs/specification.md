# 家計負担割合計算アプリ 仕様書 (Next.js & CSV連携版)

本ドキュメントは、共働き夫妻向けの「完全収入比例」による家計負担割合計算アプリの要件定義および基本設計仕様をまとめたものである。データベース（DB）を使用せず、あらかじめ用意したCSVファイルからマスタデータを読み込んで動作する、Next.jsベースの軽量かつ実用的なアプリケーションを想定している。

---

## 1. アプリケーション概要

### 1.1 背景と目的

共働き世帯において、毎月の固定費や共通の貯蓄・投資の負担割合を公平に決定することは、健全な家計運営において極めて重要である。本アプリは、月ごとに変動し得る手取り収入の比率（完全収入比例）に基づき、あらかじめ設定した共通予算に対する各自の具体的な拠出（振込）必要額をリアルタイムに自動算出することを目的とする。

### 1.2 開発スタンス

- **コアロジック**: 完全な収入比例（パターンA）
- **データ管理**: DBは一切使用せず、初期データはすべてCSVファイルから読み込む
- **技術スタック**: Next.js (App Router) + Tailwind CSS + React（Client Componentによるリアルタイム計算）

---

## 2. コア計算ロジック (完全収入比例)

本アプリの計算は、以下の3つの数式に基づいてリアルタイムに実行される。

### 2.1 変数の定義

| 変数名 | 説明 |
| :--- | :--- |
| $Income_{Husband}$ | 当月の夫の手取り収入 |
| $Income_{Wife}$ | 当月の妻の手取り収入 |
| $TotalIncome$ | 夫婦の手取り合計額（$Income_{Husband} + Income_{Wife}$） |
| $TotalBudget$ | CSVから読み込まれた、当月の共通予算（固定費・変動費・投資・貯金の総和） |

### 2.2 計算式

#### 1. 負担割合の算出

$$\text{夫の負担割合} = \frac{Income_{Husband}}{TotalIncome}$$

$$\text{妻の負担割合} = \frac{Income_{Wife}}{TotalIncome}$$

#### 2. 各自の具体的な拠出（振込）額

$$\text{夫の拠出額} = TotalBudget \times \text{夫の負担割合} \quad (\text{円未満四捨五入 or 切り捨て})$$

$$\text{妻の拠出額} = TotalBudget \times \text{妻の負担割合} \quad (\text{円未満四捨五入 or 切り捨て})$$

## 3. データ構造 (CSV仕様)

プロジェクトの `public/data/` またはサーバーサイドからアクセス可能なディレクトリに以下の構造のCSVファイルを配置する。

### 3.1 `budget_data.csv` のデータレイアウト

| 列名 (ヘッダー) | 型 | 説明 |
| :--- | :--- | :--- |
| `category` | 文字列 | `fixed_cost`（固定・変動費）、`investment`（投資・貯蓄）、`income_sample`（デフォルト収入） |
| `item` | 文字列 | 予算や項目の具体的な名称（例：家賃、共通貯金、夫基本手取り） |
| `amount` | 数値 | 金額（円単位、カンマなし） |
| `is_active` | 真偽値 | `true` の場合、初期表示時の $TotalBudget$ の合算対象とする。`false` は初期値（サンプル）扱い |

### 3.2 CSVデータの記述例

```csv
# 家計共通予算マスタデータ（コメント行は # で始める）
category,item,amount,is_active
fixed_cost,家賃,150000,true
fixed_cost,光熱費,20000,true
fixed_cost,食費,50000,true
investment,共通貯金,30000,true
investment,投資回し分(高 dividend ETF等),50000,true
income_sample,夫基本手取り,400000,false
income_sample,妻基本手取り,300000,false
```

### 3.3 初期値の自動計算ルール

- **初期予算合計 ($TotalBudget$) の自動計算**: `is_active = true` の行のみをフィルタリングして合算する。
  - 計算例: $150,000 + 20,000 + 50,000 + 30,000 + 50,000 = \mathbf{300,000円}$
- **収入のデフォルト値**: `income_sample` カテゴリの数値を、画面初期表示時の夫・妻の入力フォームにそれぞれマッピングする。

## 4. 画面構成

### 4.1 メイン画面

メイン画面は1ページ構成とし、以下のセクションで構成する。

| セクション | 内容 |
| :--- | :--- |
| 収入入力フォーム | 夫・妻それぞれの当月手取り収入を入力するフィールド（デフォルト値はCSVの `income_sample` から取得） |
| 共通予算サマリ | CSVから読み込んだ `is_active = true` の項目一覧と合計金額を表示 |
| 計算結果表示 | 夫・妻それぞれの負担割合（%）と拠出額（円）をリアルタイムに表示 |

### 4.2 リアルタイム計算

収入入力フォームの値が変更されるたびに、計算結果表示が即時更新される。ページのリロードやボタン押下は不要とする。

## 5. 技術仕様

### 5.1 ディレクトリ構成（想定）

```
project-root/
├── public/
│   └── data/
│       └── budget_data.csv       # マスタデータCSV
├── src/
│   └── app/
│       ├── page.tsx              # メインページ（Server Component）
│       └── components/
│           └── Calculator.tsx    # 計算UIコンポーネント（Client Component）
└── lib/
    └── csv.ts                    # CSV読み込みユーティリティ
```

### 5.2 データフロー

1. **サーバーサイド**: `page.tsx`（Server Component）がビルド時または初回リクエスト時に `budget_data.csv` を読み込み、パースした結果をpropsとして `Calculator.tsx` に渡す。
2. **クライアントサイド**: `Calculator.tsx`（Client Component）が収入入力フォームの状態を管理し、入力変更のたびにリアルタイムで計算式を実行して結果を更新する。

### 5.3 CSVパース仕様

- CSVのヘッダー行は必ず存在し、空行やコメント行（`#` で始まる行）を除いた最初の有効行に配置する。
- **`is_active` 列のパース**: 文字列 `"true"`（大文字・小文字を区別しないcase-insensitive比較）を `true`、それ以外（`"false"`、空文字列、`"1"` 等）を `false` として扱う。
- `amount` 列は整数として扱い、パース失敗時は `0` とする。
- 空行やコメント行（`#` で始まる行）は無視する。
